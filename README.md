# fl-web-service

Веб-сервисный слой дипломного проекта «Система классификации болезней растений
с помощью федеративного обучения в распределённой гетерогенной вычислительной
среде». Отвечает за регистрацию участников, упаковку клиента федеративного
обучения, живой дашборд, реестр обученных моделей и сервис инференса.

Само ML-ядро (модель, клиентская функция, стратегии агрегации) живёт в
отдельном репозитории и подключается как зависимость.

## Требования

- Python 3.12
- Git

Остальные инструменты (Docker, Node.js для фронтенда и т.д.) будут добавлены
в инструкцию по мере того, как в репозитории появятся соответствующие части.

## Быстрый старт

### 1. Клонирование репозитория

```bash
git clone https://github.com/nxhxl000/fl-web-service.git
cd fl-web-service
```

### 2. Виртуальное окружение

Создаётся один venv на весь репозиторий, в папке `.venv` в корне. Эта папка
игнорируется git'ом.

```bash
python3 -m venv .venv
```

Активация:

```bash
# Linux / macOS
source .venv/bin/activate

# Windows (PowerShell)
.venv\Scripts\Activate.ps1
```

Деактивация в любой момент:

```bash
deactivate
```

### 3. Зависимости

На текущем этапе разработки зависимости ещё не зафиксированы — они будут
добавляться в `pyproject.toml` по мере появления кода бэкенда и клиента.
Сразу после создания venv устанавливать ничего не нужно.
